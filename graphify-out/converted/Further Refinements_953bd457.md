- In the issue log, The title of each tile seems to have some character limit as titles are looking like this:
“Credit card search result: After searching for 'credit card', Zaid immediately landed on a cold...”
“Menu tile navigation: Zaid scrolled through several dense and visually overloaded menu tiles without...”
“No clear upgrade CTA: After starting the EMI card activation, Zaid was not shown any next steps or a...”
Instead of this the title should be framed more from feature/element etc kind of product manager language without persona names. Also it should look like a title. Take inspiration from how people give title to a user story in Azure Boards. It should carry that kind of flavour.
- In the issue log, within each issue tile, immediately below the title the detailed H2 or H3 line should be again not include persona names but briefly capture and summarize the issue and recommendation properly.
- In issue log tab, there should also be a sorting and filtering functionality based on the recency of issue/recommendation tile creation. Remove source field. There should also be a functionality to permanently delete a issue/recommendation tile.
- On the dashboard tab, there should also be a functionality to filter based on product/business tag wise tiles through a drop down with functionality to select multiple options.
- Additional Functionality such that the user can choose to type persona details via a frontend section within run audit tab so that the user can enter persona details in plain English via some bare minimum guided fields. There should be functionality to add more than one persona via a simple ‘+’ button and then type in plain english. The functionality to upload the persona details via .md file should also remain. Either of them must be mandatory to run the audit.
- Possibility of selecting device viewport or auto-selecting the latest/popular model.
- Sim binding functionality
- Access Denied is still coming up via search.
- Cost view in INR for each persona run on each persona tile. IT should be able to categorize properly in case of multiple persona run in one single go.
